# Privacy Policy Draft

## Summary

Žvaka stores meal tracking data primarily on the device. The app can optionally back up app data to the user's Google Drive in the app-private Drive storage area.

## Data Collected And Stored

- Profile information entered by the user:
  first name, age, height, weight, sex, activity level
- Meal tracking data:
  food photos, calorie estimates, manual calorie entries, meal labels, timestamps
- Coach conversation history stored locally on the device
- App preferences:
  onboarding completion, coaching settings, selected estimation mode

## How Data Is Used

- To calculate a daily calorie budget
- To estimate calories from food photos on-device
- To display meal history, summaries, and trends
- To support optional coach conversations based on local meal data
- To provide optional Google Drive backup and restore

## Google Drive Backup

- Google Drive backup is optional and user-initiated
- Backups are stored in the app's hidden Google Drive `appDataFolder`
- Backups include profile data, calorie budget history, meal history, coach chats, app preferences, and saved meal photos
- Downloaded local AI model files are not backed up to Google Drive

## Data Sharing

- The app does not sell user data
- The app does not upload meal data to a cloud AI API for calorie estimation
- Google Drive backup uses the user's Google account only when the user enables backup

## Data Retention And Deletion

- Users can erase local app data from inside the app
- Users can disable Google Drive sync from inside the app
- Restoring from Google Drive replaces current local data with the selected backup state
